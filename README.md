# Медицинский портал
Внутренний портал для сотрудников ГБУЗ РБ ГДКБ №17 г. Уфа.

Проект не является SaaS. Это закрытая внутренняя система для одной больницы.

---

# 🎯 Назначение системы

Портал предназначен для:

- Публикации объявлений
- Размещения документов
- Подачи заявок (ИТ, АХЧ, метролог)
- Сообщения о нежелательных событиях (инцидентах)
- Управления справочниками (отделения, виды нежелательных событий)

# 📋 Форма "Нежелательное событие"

Форма публичная. Авторизация НЕ требуется.

После отправки данные сохраняются в БД со статусом NEW.

## Поля формы:

- Дата события
- Время события
- Место
- ФИО пациента
- Дата рождения пациента
- Обстоятельства
- ФИО сотрудника
- Должность сотрудника
- Присутствие законного представителя (select)
- Отделение (select из БД)
- Вид нежелательного события (select из БД)
- Последствия

---

# 🔐 RBAC (Role-Based Access Control)

Доступ к разделам определяется ролью пользователя.

## Роли:

- Главный врач
- Администратор
- Контроль качества
- ИТ отдел
- АХЧ отдел
- Metrolog
- Сотрудник

---

## Принцип:

Backend проверяет роль через middleware.

Пример:

- Только Администратор может управлять справочниками
- Контроль качества работает с инцидентами
- ИТ отдел видит только ИТ заявки

Frontend скрывает интерфейс на основе роли,
но основная защита всегда на Backend.

---

# 🧾 Администрирование

Администратор может:

- Добавлять отделения
- Добавлять виды нежелательных событий

Форма инцидента НЕ редактируется администратором.
Она фиксирована в коде.

# 🌐 API Принципы

- Все API начинаются с /api
- Использовать параметризованные запросы ($1, $2)
- Никаких raw SQL с подстановкой строк
- Всегда release() после pool.connect()
